TASK
Evaluate if a Canadian education program (CIP) is an OPTIMAL pathway into a Canadian occupation (NOC).

OPTIMAL = CIP teaches the core technical / professional / regulatory competencies normally required for entry, with minimal retraining beyond onboarding.

RULES (must follow)
- Output exactly ONE label: YES | LEAN_YES | UNSURE | LEAN_NO | NO
- Use only provided CIP/NOC info (do not assume facts not present)
- Generic skills (communication, critical thinking, leadership, teamwork, “business skills”, etc.) never justify YES/LEAN_YES
- You MUST anchor the decision primarily on NOC employment_requirements (then main_duties). If the CIP does not support entry requirements described there, you cannot output YES/LEAN_YES.
- You MUST use the full scale: if it is clearly NOT a pathway, choose NO (not LEAN_NO).

HARD GATE (overrides everything)
If NOC employment_requirements mention ANY entry gate such as:
- apprenticeship, trade certification, licensing, registration, certification exam, regulated occupation, required credential, board exam
THEN you must output LEAN_NO or NO unless the CIP text explicitly indicates it is that credential/training pathway.
- Do NOT assume a CIP implies a licence/apprenticeship unless the CIP text explicitly says so.
- If the NOC is clearly credential-gated and CIP does not indicate the pathway, choose NO (not LEAN_NO), with confidence 0.05–0.12.

LABEL GUIDE (decision boundaries)
YES:
- CIP is a common/standard direct entry pathway for this NOC AND
- CIP covers the core competencies AND supports the education/credential expectations described in NOC employment_requirements.
- Only minor onboarding / orientation needed.

LEAN_YES:
- CIP aligns with BOTH:
  (a) NOC main_duties AND
  (b) the education/credential expectations in NOC employment_requirements.
- Any gap is minor AND not an entry gate (e.g., a specific tool, a narrow context, a short bridging course).
- If the gap relates to apprenticeship/licensing/certification/registration, LEAN_YES is NOT allowed.

UNSURE:
- Provided info is insufficient or ambiguous to determine alignment (missing key details or unclear scope).
- Use UNSURE when employment_requirements or CIP definition/examples are too vague to judge entry readiness.

LEAN_NO:
- Some meaningful overlap (adjacent domain), but IMPORTANT gaps exist:
  - missing multiple core duties OR
  - needs substantial retraining OR
  - employment_requirements suggest preparation not supported by CIP text (but not a clear “obvious mismatch” case).
- Use LEAN_NO when overlap is real but still not a viable entry pathway without significant upskilling.

NO (use this more often than LEAN_NO when clearly wrong):
Choose NO if ANY of the following are true based on the provided text:
- Clear domain mismatch with little/no overlap in core duties (e.g., policing vs carpentry; telecom management vs financial advising; atmospheric chemistry vs translators; engineering vs unrelated arts/humanities; clinical medicine vs unrelated trades).
- NOC is credential/licence/apprenticeship-gated AND CIP does not indicate that credential/training pathway (see HARD GATE).
- CIP’s definition/exclusions indicate the program is not preparing for the NOC’s core work.
- CIP prepares for a fundamentally different profession than the NOC (not just “missing a course”).

CONFIDENCE (JSON number 0–1, 2 decimals max)
- Confidence must be consistent with the label range:
  YES ≥0.80
  LEAN_YES 0.60–0.79
  UNSURE 0.40–0.59
  LEAN_NO 0.20–0.39
  NO ≤0.19

Calibration rules:
- Do NOT default to the same confidence repeatedly. Pick a value that reflects strength of evidence.
- Never reuse the same confidence value more than 3 times in a row across answers; vary within the label range based on evidence strength.
- For NO:
  - 0.05–0.12 when obvious domain mismatch or credential-gated mismatch
  - 0.13–0.19 when mostly mismatch but a small ambiguous overlap exists
- For LEAN_NO:
  - 0.20–0.29 when overlap is weak
  - 0.30–0.39 when overlap is moderate but still not a viable entry pathway
- If substantial retraining is required, confidence must be ≤0.59.

OUTPUT
Return ONLY JSON with keys:
label, confidence, rationale, model_version, prompt_version

rationale: 1–2 short sentences that mention:
- at least 1 CIP detail (definition/examples/exclusions/title) AND
- at least 1 NOC detail from employment_requirements (preferred) or main_duties.
If NOC employment_requirements are empty or not informative, explicitly say so.
Do not cite generic skills.

model_version: short string
prompt_version: "{{prompt_version}}"
